# Customer Shopping Behavior Analysis

## Overview

This project focuses on analyzing customer shopping behavior using Python, PostgreSQL, and Power BI. The workflow includes data loading, exploratory data analysis (EDA), data cleaning, SQL-based analysis, dashboard creation, and reporting.

The goal of the project is to identify customer trends, purchasing patterns, and business insights that can support data-driven decision-making.


## Dataset

The dataset contains customer shopping behavior information such as:

* Customer demographics
* Purchase history
* Product categories
* Payment methods
* Shopping frequency
* Spending patterns

The dataset was analyzed and cleaned before being used for SQL querying and dashboard development.

## Project Workflow

### 1. Data Loading

* Imported dataset into Python using Pandas
* Checked data structure and basic information

### 2. Exploratory Data Analysis (EDA)

Performed analysis to understand:

* Missing values
* Data distribution
* Customer purchasing behavior
* Sales trends
* Category-wise insights

### 3. Data Cleaning

* Removed duplicates
* Handled missing values
* Standardized column formats
* Corrected inconsistent data

### 4. SQL Analysis (PostgreSQL)

Executed SQL queries to:

* Analyze customer segments
* Calculate sales metrics
* Identify top-performing categories
* Generate business insights

### 5. Power BI Dashboard

Built an interactive dashboard with:

* KPIs
* Sales trends
* Customer segmentation
* Product category analysis
* Interactive filters and charts
